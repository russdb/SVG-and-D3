SVG

An introduction to data presentations

SVG, or Scalable Vector Graphics, is an XML-based vector image format. It has been an open standard, developed by the W3C since 1999, yet it is still relatively unknown. Because it is derived from XML, the images, and their corresponding behaviors, can be controlled by CSS and Javascript. And while SVG is relatively simple in theory, some imagination along with the animation, container, gradient, and graphical elements your images can be as complex and flexible as you need.

Usage

Start by using an <svg>tag. <svg> is similar to <div>, but for simple graphics such as logos, charts, etc.

heads up! In theory, the svg canvas is infinite, it is a good idea to place a width and height attribute on it!

</svg>

Including SVG's from External Sources

While you can code SVG inline, it is much better practice to include it in a seperate file, much like you would CSS or Javascript there are a few different ways, and each has its advantages and drawbacks, but the <object> tag is considered the best practice

See also:
  • W3C Shapes

Types of Sources

These include: <img>, <object>, <embed>, and the <iframe> tags 

<image src="image.svg" />

Heads up! This is not preferred, or recommended by the w3c.. it is not compliant across all browsers, and cannot be manipulated by SMIL, CSS or Javascript! 


<embed src="chart.svg" type="image/svg+xml" width="66%" />

Heads up! This is not preferred, or recommended by the w3c.. it is not compliant across all browsers, and cannot be manipulated by SMIL, CSS or Javascript! 

                                    
<iframe src="images/JSLogo.svg" 
           scrolling="no" 
           style="border: none">
    <-- older browser fall back! -->
    <img src="images/JSLogo.png" alt="Fallback image" />
</iframe>

Heads up! While we are afforded a fall back, it can be quite tricky to script for an iframe. Furthermore, this tag has never been formally approved, although approval is coming soon. It is also very dependent heavy, and you can get different results depending upon the browser 

                                    
<object id="object-image" 
        type="image/svg+xml" 
        data="images/MSLogo.svg"
        width="75%" 
>

    <!-- this is for IE, and allows for full scripting capabilities in all 5 major browsers -->
    <param name="src" value="images/MSLogo.svg">
</object>

Heads up! This is the preferred and recommended way to embed an svg tag, and its use affords certain advantages:
  • Fall back for older browsers
  • Supported by all major browsers
  • perfect for hooking into CSS and javascript
  • if it must be animated, it must be <object> or an <iframe> (yuck!)

Container Elements

SVG includes a host of container elements for use in graphics of all kinds. They are the building blocks of SVG, they are <a> <defs> <g> <marker> <mask> <missing-glyph> <pattern> <switch> and <symbol>

Element Descriptions These are just a few, for a full list, click here

<g>

The ‘g’ element is a container element for grouping together related graphics elements. If a graphical element is not contained in a grouping it is conceptually reagarded to be in its own group. 

                                        
<svg width="500" height="275" viewBox="0 0 95 50"">

  <g stroke="green" fill="white" stroke-width="5">
    <circle cx="25" cy="25" r="15"/>
    <circle cx="40" cy="25" r="15"/>
    <circle cx="55" cy="25" r="15"/>
    <circle cx="70" cy="25" r="15"/>
  </g>

</svg>
More Info

<defs>

SVG allows graphical objects to be defined for later reuse. It is recommended that, wherever possible, referenced elements be defined inside of a <defs> element. Defining these elements inside of a <defs> element promotes understandability of the SVG content and thus promotes accessibility. Graphical elements defined in a <defs> element will not be directly rendered. You can use a <use> element to render those elements wherever you want on the viewport. 

                                        
<svg width="500" height="275" viewBox="0 0 500 275">

  <defs>
    <linearGradient id="Gradient01">
      <stop offset="20%" stop-color="#39F" />
      <stop offset="90%" stop-color="#F3F" />
    </linearGradient>
  </defs>

  <rect x="10" y="10" width="500" height="200" 
    fill="url(#Gradient01)" />

</svg>
More Info

<symbol>

The <symbol> element is used to define graphical template objects which can be instantiated by a <use> element. The use of symbol elements for graphics that are used multiple times in the same document adds structure and semantics. Documents that are rich in structure may be rendered graphically, as speech, or as Braille, and thus promote accessibility. Note that a symbol element itself is not rendered. Only instances of a symbol element (i.e., a reference to a symbol by a <use> element) are rendered. 

                                        
<svg width="500" height="275" viewBox="0 0 500 275">

  <symbol id="sym01" viewBox="0 0 150 110">
    <circle cx="50" cy="50" r="40" stroke-width="8" stroke="red" fill="red"/>
    <circle cx="90" cy="60" r="40" stroke-width="8" stroke="green" fill="white"/>
  </symbol>

  <!-- actual drawing by "use" element -->
  <use href="#sym01" x="0" y="0" width="100" height="50"/>
  <use href="#sym01" x="0" y="50" width="75" height="38"/>
  <use href="#sym01" x="0" y="100" width="50" height="25"/>

</svg>
